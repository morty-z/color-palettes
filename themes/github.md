# GitHub

GitHub's markdown style is the de facto standard for developer documentation, inspired by GitHub's README and issue rendering. It pairs a neutral grayscale palette with a signature blue link color and uses bottom-bordered headings to create clear content hierarchy. Ideal for open-source projects, technical docs, and developer-facing content.

- **Mode**: light
- **Font category**: sans-serif
- **Best for**: README files, technical documentation, developer blogs, open-source projects
